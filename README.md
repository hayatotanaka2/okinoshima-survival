# 僕らのサマーウォーズ ver-B

同期旅行で使う、スマホ向けPWAのサバイバルゲーム管理アプリです。

## 技術スタック

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase
- localStorage fallback
- PWA

## セットアップ

```sh
npm install
npm run dev
```

## 起動

```sh
npm run dev
```

ブラウザで `http://localhost:3000` を開きます。

## 管理者

管理者画面は `/admin` です。

## ゲームの流れ

1. 参加者はトップ画面で自分の名前を選ぶ。
2. 管理者画面でメンバーを確認・追加する。
3. 管理者画面でランダム編成案を作成し、必要なら手動調整して確定する。
4. 管理者がミッションを発動する。
5. チームが達成したら管理者が達成承認する。
6. 報酬沖コインがメンバーへ均等配分される。
7. 宝箱コード、物資、オークションで沖コインを使って盛り上げる。
8. 士気掲示板で波乗れてない行動を告発し、裁判所で幹事が判決を出す。

## 自分選択と所有物

ログイン機能は使わず、端末ごとに「この端末のプレイヤー」を選びます。

選択したメンバーIDはブラウザのlocalStorageに保存されます。沖コイン、所持物資、チーム、ミッションなどの本体データはSupabaseのGameStateに保存され、全員に同期されます。

一度選択した後は、画面右上のプロフィールボタンから変更できます。初回以外はホームの大きな選択欄を出さないため、現地での誤操作を減らします。

物資・カードは原則として個人所有です。チームで購入したオークション景品も、落札処理時に管理者が選んだ受け取りメンバーの所持物資として登録されます。

秘匿カードは、所持している本人だけが本当のカード名と説明を確認できます。未所持者には「宝箱Aカード」のような表向き名だけが表示されます。ミッション報酬でチーム全員に配る場合も、内部的には各メンバーへ同じ秘匿カードを1枚ずつ配布します。

## チーム編成

チーム編成は管理者画面でのみ実行できます。

管理者はチーム数を指定してランダム編成案を作成し、確認画面でメンバーを別チームへ移動してから確定します。確定時に全端末へ同期され、イベントログと最新通知に「ランダム編成しました」と表示されます。

幹事（はやと、まさこ、あやなん、こひ）をできるだけ別チームへ分散するチェックも用意しています。

チームタブの物資・カードと達成ミッションは、現在のチーム結成後に獲得・達成したものだけを表示します。チーム再編成後は新規チーム扱いになり、その欄はなし表示から始まります。個人の所持物資自体はメンバー画面に残ります。

## PWAとして使う

iPhoneではSafariでURLを開き、共有ボタンから「ホーム画面に追加」を押します。追加後はホーム画面のアイコンから起動します。

起動時には参加メンバーのアバターが集まる短い演出が表示されます。毎回出ると重いため、同じ端末では一定時間に1回だけ表示します。通常画面では、背景に一部メンバーのアバターが薄く漂います。

## iPhone確認

Macで以下を実行します。

```sh
npm run dev -- --host 0.0.0.0
ipconfig getifaddr en0
```

iPhoneを同じWi-Fiに接続し、Safariで `http://192.168.x.x:3000` を開きます。

## 全員同期

Vercel本番で全員が同じゲーム状態を見るにはSupabase設定が必須です。

詳しい手順は [docs/SUPABASE_SETUP.md](docs/SUPABASE_SETUP.md) を見てください。
実作業のチェックリストは [docs/DEPLOYMENT_CHECKLIST.md](docs/DEPLOYMENT_CHECKLIST.md) です。

必要なVercel環境変数:

```txt
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
```

Supabase未設定の場合はlocalStorage fallbackで動きますが、その場合は端末ごとに別データになります。

## 通知

MVPではアプリ内通知をGameStateに保存して表示します。Supabase設定済みなら全員に同期されます。Web Push通知は将来拡張用の雛形のみ用意しています。

## ホーム画面

ホーム画面には最新通知3件、各画面へのリンク、スクロール式イベントログ、PWA案内を表示します。写真投稿は管理者ボタンの横にある「写真投稿」から開けます。緊急ミッション通知はビビッドピンクで強調表示されます。

## 士気掲示板

下部ナビゲーションの「士気」から、告発掲示板と裁判所に入れます。掲示板では参加者が士気下げ行動を告発できます。裁判所は管理者パスコードで入り、告発ごとにGUILTYまたはNOT GUILTYを判定します。判決時は全員の画面に判決エフェクトが表示されます。

## Vercelデプロイ

このアプリはVercelで動かす前提のNext.jsアプリです。

1. GitHubにこのフォルダの内容をpushする。
2. VercelでNew Projectを選ぶ。
3. GitHub repositoryを選択する。
4. Framework PresetはNext.jsを選ぶ。
5. Build Commandは通常どおり `npm run build`。
6. Output Directoryは空欄のままでよい。
7. Deployする。

Vercel側では `package.json` を見て依存関係をインストールし、Next.jsとしてビルドします。Mac側に入る `node_modules` はVercelにはアップロードしません。

### デプロイ前チェック

ローカルでNode.js/npmが使える場合は、push前に以下を実行してください。

```sh
npm install
npm run build
```

ローカルにNode.js/npmを入れずに進める場合でも、GitHubへpushすればVercel側でインストールとビルドが走ります。ただし、エラーが出た場合はVercelのBuild Logsを見て修正します。

### 注意

MVPの本番データ保存はSupabaseです。Supabase環境変数がない場合だけlocalStorage fallbackで動きます。

## 今後のデータ設計方針

現在は旅行直近のMVPとして、Supabaseの `game_states` テーブルにGameState全体をJSONBで保存します。

今後より堅くする場合は、`members`, `teams`, `missions`, `items`, `event_logs` を個別テーブルに分け、管理者操作はVercel API Route経由でサーバー側検証します。
