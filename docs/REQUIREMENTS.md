# REQUIREMENTS.md

## 1. アプリ概要

「沖ノ島サバイバル」は、館山・沖ノ島で21人程度の大人数旅行を行う際に使用する、スマホ向けPWAです。

参加者はチームに分かれてミッション、物資探索、オークションを行い、旅行全体をゲーム化します。幹事は管理者画面から、チーム編成、ミッション発動、達成承認、コイン付与、アイテム付与、ログ確認を行います。

## 2. 想定ユーザー

- 幹事
- 参加者
- 参加人数: 23人
- チーム数: 3から4チーム程度
- 1チーム: 5から7人程度

## 3. 技術スタック

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase
- localStorage fallback
- PWA

## 4. MVPの目的

旅行当日に全員が同じゲーム状態を見られるアプリを完成させます。

Vercel本番ではSupabaseをメイン保存先にします。Supabase未設定時だけlocalStorage fallbackで動かします。

MVPではGameState全体をSupabaseのJSONB 1行に保存します。将来的にはmembers、teams、missionsなどを個別テーブルに正規化します。

## 5. 主要機能

### メンバー管理

- メンバーを登録できる。
- メンバーの名前、所持沖コイン、所持アイテム、所属チーム、累計獲得コイン、累計消費コインを管理できる。
- 初期メンバーは今回参加する23人を登録済みにする。
- 参加者は端末ごとに自分の名前を選択し、その選択を端末に保存できる。

### チーム編成

- チーム数を指定できる。
- 管理者画面でランダムチーム分けできる。
- 管理者画面でイベントごとにチーム再編成できる。
- ランダム生成後、管理者が確認画面でメンバーを手動移動してから確定できる。
- 幹事（はやと、まさこ、あやなん、こひ）をできるだけ別チームへ分散して編成できる。
- 現在のチームごとの人数と合計沖コインを表示できる。
- 手動調整はMVPでは簡易でよいが、将来的に追加しやすい設計にする。

### 沖コイン

- 沖コインは個人が持つ。
- チームの強さは所属メンバーの沖コイン合計で計算する。
- チーム報酬はメンバーに均等配分する。
- チーム支払いはMVPでは人数割りにする。

### ミッション

- ミッション一覧を表示できる。
- 管理者がミッションを発動できる。
- 管理者が緊急発動すると、全端末に一時的な緊急ミッション演出を表示できる。
- 管理者が達成承認できる。
- 達成時にチームまたは個人へコインやアイテムを付与できる。

### 物資・アイテム

- 物資一覧を表示できる。
- 管理者がアイテムを作成・付与できる。
- 物資・カードは原則として個人所有にする。
- 参加者も自分のアイテムを使用できる。
- アイテム使用時は状態をusedにし、イベントログと通知に残す。
- アイテムは景品、便利効果、文明カード、妨害カードに分類する。

### オークション

- 夜のオークション景品を表示できる。
- 管理者が落札チームと落札価格を設定できる。
- オークション開始・終了を管理できる。
- 参加者側から入札できる。
- 現在最高額と最高入札チームを共有表示できる。
- 落札価格をチームメンバーから人数割りで減算できる。
- 落札品は管理者が選んだ受け取りメンバーの所持物資として登録できる。
- 落札処理はイベントログに残す。

### QR宝箱

- MVPではQR読み取りが難しければ宝箱コード入力方式でよい。
- 管理者が宝箱コードを作成できる。
- 参加者がコードを入力して報酬を受け取れる。
- 一度使われたコードは再使用できない。

### イベントログ

- 重要操作を時系列で表示する。
- ミッション達成、QR宝箱発見、オークション落札、コイン増減、システム操作を記録する。

### アプリ内通知

- 通知をGameStateに保存する。
- トップ画面に最新通知を表示する。
- 管理者が通知を作成できる。

### 管理者画面

- 簡易パスコード認証を行う。
- パスコードはMVPでは `admin123` とする。

管理者ができること:

- メンバー追加
- メンバー削除
- メンバーの沖コイン加算・減算
- チーム作成
- チーム削除
- ランダムチーム編成
- ランダムチーム編成案の確認・手動調整・確定
- チーム再編成
- ミッション作成
- ミッション発動
- ミッション終了
- ミッション達成承認
- アイテム作成
- アイテム付与
- オークション景品作成
- オークション落札処理
- 宝箱コード作成
- 通知作成
- イベントログ確認
- ゲームリセット

### 写真投稿・ミッション証跡

- 参加者がミッション達成写真を投稿できる。
- 投稿にはミッション、チーム、投稿者、画像URL、コメント、承認状態を持たせる。
- 管理者が投稿を確認して承認できる。
- MVPでは画像保存先はSupabase Storageを想定する。

## 6. PWA要件

- manifest.jsonを設定する。
- displayはstandaloneにする。
- theme_colorとbackground_colorを設定する。
- アイコンを設定する。
- service workerを用意する。
- iPhoneのホーム画面に追加できる構成にする。

## 7. Supabase同期要件

- 全員で同じゲーム状態を共有する。
- Vercel本番ではSupabaseを使用する。
- localStorageはfallbackと端末キャッシュとしてのみ使う。
- Supabase Realtimeで他端末の更新を反映する。
- MVPでは `game_states` テーブルに `id = main` の1行を保存する。
- 写真投稿を実装する場合はSupabase Storageを使う。

## 8. MVP完成条件

- npm run devで起動できる。
- npm run buildが通る。
- スマホ幅でUIが崩れない。
- メンバーを登録できる。
- チーム数を指定してランダム編成できる。
- チーム再編成できる。
- 個人ごとの沖コインを表示できる。
- チーム合計沖コインを表示できる。
- 管理者がコインを増減できる。
- ミッションを発動できる。
- ミッション達成でコインを付与できる。
- ミッション達成写真を投稿・承認できる。
- アイテムを付与できる。
- 参加者がアイテムを使用できる。
- オークション落札処理ができる。
- 参加者がオークションに入札できる。
- 宝箱コードを作成・入力できる。
- イベントログが残る。
- アプリ内通知が表示される。
- Supabase設定時に全端末でGameStateが共有される。
- manifest.jsonが設定されている。
- ホーム画面追加できるPWA構成になっている。
