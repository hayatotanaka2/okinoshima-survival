# UI_SPEC.md

## 1. 共通方針

- スマホファーストで実装する。
- 屋外で操作するため、ボタンは大きめにする。
- 下部ナビゲーションを用意する。
- 管理者画面以外は参加者が見る前提で、情報を見やすくする。
- 管理者画面は操作ミスを避けるため、セクションごとに整理する。

## 2. トップ画面

パス: `/`

表示内容:

- アプリ名「沖ノ島サバイバル」
- 現在のゲーム状態
- チーム一覧
- チームごとの合計沖コイン
- ランキング
- 最新通知
- 最新イベントログ
- 各画面へのリンク

リンク:

- メンバー
- チーム
- チーム編成
- ミッション
- 物資
- オークション
- QR宝箱
- 写真投稿
- 管理者画面

## 3. メンバー画面

パス: `/members`

表示内容:

- メンバー一覧
- 名前
- 所持沖コイン
- 所属チーム
- 所持アイテム
- 累計獲得コイン
- 累計消費コイン

## 4. チーム画面

パス: `/teams`

表示内容:

- チーム一覧
- チーム名
- チームカラー
- メンバー一覧
- チームポイント
- チーム合計沖コイン
- チーム所有アイテム
- 達成済みミッション

## 5. チーム編成画面

パス: `/teams/compose`

機能:

- チーム数を指定する。
- ランダムチーム分けを実行する。
- 現在のチーム構成を表示する。
- チームごとの人数を表示する。
- チームごとの合計沖コインを表示する。

MVPでは手動ドラッグ移動は不要です。必要なら管理者画面で簡易調整を後から追加します。

## 6. ミッション画面

パス: `/missions`

表示内容:

- ミッションカード一覧
- タイトル
- 説明
- 難易度
- 報酬ポイント
- 報酬沖コイン
- 報酬アイテム
- 状態
- 達成済みチーム

参加者側では閲覧中心です。達成承認は管理者画面で行います。

## 7. 物資画面

パス: `/items`

表示内容:

- アイテムカード一覧
- アイテム名
- 説明
- 種類
- 価値
- 所有者
- 状態

## 8. オークション画面

パス: `/auction`

表示内容:

- 景品カード一覧
- 景品名
- 説明
- 現在価格
- 落札チーム
- 状態

MVPでは参加者によるリアルタイム入札は不要です。管理者が落札処理を行います。

## 9. QR宝箱画面

パス: `/qr`

MVPでは宝箱コード入力画面として実装します。

表示・機能:

- 宝箱コード入力フォーム
- 取得者メンバー選択
- 取得チーム表示または選択
- 報酬受け取りボタン
- 取得済み宝箱一覧

将来的にQR読み取り機能を追加できる構造にします。

## 10. 管理者画面

パス: `/admin`

認証:

- パスコード入力。
- MVPのパスコードは `admin123`。

セクション:

- メンバー管理
- コイン管理
- チーム編成
- ミッション管理
- アイテム管理
- オークション管理
- QR宝箱管理
- 通知作成
- イベントログ確認
- ゲームリセット

## 12. 写真投稿画面

パス: `/submissions`

機能:

- ミッションを選択する。
- チームを選択する。
- 投稿者を選択する。
- 写真を投稿する。
- コメントを添える。
- 投稿状況を一覧表示する。

管理者画面で承認されると、ミッション達成処理と報酬配布が行われます。

管理者操作はすべてイベントログに残します。

## 11. 下部ナビゲーション

主要リンク:

- ホーム
- チーム
- ミッション
- 物資
- 管理

必要に応じて、その他画面はトップ画面のリンクから遷移します。
