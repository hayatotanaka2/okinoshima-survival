# UI_SPEC.md

## 1. 共通方針

- スマホファーストで実装する。
- 屋外で操作するため、ボタンは大きめにする。
- 下部ナビゲーションを用意する。
- 管理者画面以外は参加者が見る前提で、情報を見やすくする。
- 管理者画面は操作ミスを避けるため、セクションごとに整理する。

## 2. トップ画面

パス: `/`

表示内容:

- アプリ名「沖ノ島サバイバル」
- 現在のゲーム状態
- 最新通知
- 最新イベントログ
- 各画面へのリンク
- この端末のプレイヤー選択

リンク:

- メンバー
- チーム
- チーム編成
- ミッション
- 物資
- オークション
- QR宝箱
- 写真投稿
- 管理者画面

## 3. メンバー画面

パス: `/members`

表示内容:

- メンバー一覧
- 名前
- 所持沖コイン
- 所属チーム
- 所持アイテム
- 累計獲得コイン
- 累計消費コイン

## 4. チーム画面

パス: `/teams`

表示内容:

- チーム一覧
- チーム名
- チームカラー
- メンバー一覧
- チーム合計沖コイン
- 所属メンバーの所持アイテム
- 達成済みミッション

表示する所持アイテムと達成済みミッションは、現在のチーム結成後に獲得・達成したものに限定します。チーム再編成後は新規チーム扱いになり、物資・カード・達成ミッション欄はなし表示から始まります。

## 5. チーム編成画面

パス: `/teams/compose`

機能:

- 参加者側からはチーム編成を実行できない。
- 管理者画面への案内のみ表示する。

チーム編成の実行、確認、手動調整、確定は管理者画面で行います。

## 6. ミッション画面

パス: `/missions`

表示内容:

- ミッションカード一覧
- タイトル
- 説明
- 報酬沖コイン
- 報酬アイテム
- 状態
- 達成済みチーム
- 自分の現在チームが達成済みか未達成か

参加者側では閲覧中心です。達成承認は管理者画面で行います。

ミッション自体は全チームに発動されます。あるチームが達成承認されてもミッション全体は閉じず、他チームは引き続き写真投稿と達成承認の対象になります。

管理者が緊急発動した場合は、全端末で短時間の緊急ミッションモーダルを一度表示し、その後は通常のミッション一覧へ戻ります。

## 7. 物資画面

パス: `/items`

表示内容:

- アイテムカード一覧
- アイテム名
- 説明
- 種類
- 価値
- 所有者
- 状態
- 端末に保存した自分選択を初期値として使う。
- 使用できるのは自分の所有物資を基本とする。

## 8. オークション画面

パス: `/auction`

表示内容:

- 景品カード一覧
- 景品名
- 説明
- 現在価格
- 落札チーム
- 落札品の受け取りメンバー
- 状態

参加者は自分を選ぶと所属チームで入札できます。管理者は落札処理時に受け取りメンバーを選びます。

## 9. QR宝箱画面

パス: `/qr`

MVPでは宝箱コード入力画面として実装します。

表示・機能:

- 宝箱コード入力フォーム
- 取得者メンバー選択
- 取得チーム表示または選択
- 報酬受け取りボタン
- 取得済み宝箱一覧
- 取得者は端末に保存した自分選択を初期値にする。

将来的にQR読み取り機能を追加できる構造にします。

## 10. 管理者画面

パス: `/admin`

認証:

- パスコード入力。
- MVPのパスコードは `admin123`。

セクション:

- メンバー管理
- コイン管理
- チーム編成
- チーム編成案の確認・手動移動・確定
- ミッション管理
- アイテム管理
- オークション管理
- QR宝箱管理
- 通知作成
- イベントログ確認
- ゲームリセット

## 12. 写真投稿画面

パス: `/submissions`

機能:

- ミッションを選択する。
- チームを選択する。
- 投稿者を選択する。端末に保存した自分選択を初期値にする。
- 写真を投稿する。
- コメントを添える。
- 投稿状況を一覧表示する。

管理者画面で承認されると、ミッション達成処理と報酬配布が行われます。

管理者操作はすべてイベントログに残します。

## 11. 下部ナビゲーション

主要リンク:

- ホーム
- チーム
- ミッション
- 物資
- 管理

必要に応じて、その他画面はトップ画面のリンクから遷移します。
