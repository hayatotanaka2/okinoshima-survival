# IMPLEMENTATION_SCOPE.md

## 必須実装

以下は旅行本番までに必ず実装します。

1. Supabase本番同期
2. ミッション作成・編集・削除・終了
3. メンバー削除、名前変更、手動チーム移動、チーム名変更、チーム削除
4. 参加者によるアイテム使用
5. 参加者によるオークション入札
6. Web Push通知または通知許可までの実運用確認
7. 写真投稿・管理者承認
8. Vercelデプロイと実URL確認

## できれば実装

- QRコード生成
- QRコード読み取り
- 印刷用QR一覧

MVPでは宝箱コード入力でも運用可能です。

## 実装しない

- チーム役職
- 役職能力

## セキュリティ方針

旅行直近MVPでは、URLを知っている同期メンバーだけが使う前提で、Supabase anon keyによる共有GameState更新を許容します。

ただし、以下の条件に当てはまる場合は、管理者操作をVercel API Route経由に変更します。

- 参加者側の操作範囲を厳密に制限したい
- URLが外部に広がる可能性がある
- コインや報酬の改ざんリスクを下げたい
- 旅行後も継続運用する

本番直近では、管理者画面のパスコードを強めることを優先します。
