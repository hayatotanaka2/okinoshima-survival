1. 今回作るアプリの全体像

今回のアプリは、単なる「チーム対抗ポイント管理アプリ」ではなく、次のようなゲームです。

コンセプト

個人が沖コインを持ち、イベントごとにチームが再編される、旅行全体をゲーム化するPWA

です。

つまり、ポイントやコインをチームだけに持たせるのではなく、各個人が沖コインを持つ設計にします。

これが大事です。

なぜなら、今回の旅行ではイベントごとにチームが変わる可能性があるからです。

例えば、昼の沖ノ島サバイバルでは赤チームだった人が、夜のゲームでは青チームになるかもしれません。
このとき、チームだけにポイントを持たせると、再編成した瞬間にスコア管理が破綻します。

なので、

個人が沖コインを持つ
↓
イベントごとにチームを編成する
↓
チームの強さは所属メンバーの沖コイン合計で決まる
↓
チームで勝った報酬は個人に配分される
↓
チームで支払った費用は個人から按分して引かれる

という構造にします。

2. ゲーム設計の整理

Member {

重要なのは、沖コインは個人に紐づくことです。

2.2 チーム

チームはイベントごとに作られます。

チーム自体が持つ情報は以下です。

Team {
  id: string
  name: string
  color: string
  memberIds: string[]
  roleAssignments: RoleAssignment[]
}

チームの合計コインは、DBに保存するというより、毎回メンバーの合計から計算します。

teamCoin = team.memberIds.map(member.coin).sum()

こうすると、チーム再編成しても自然です。

# 要件定義書：沖ノ島サバイバル PWA MVP

## 1. プロジェクト概要

Next.jsで、スマホ向けPWAとして動作する「沖ノ島サバイバルゲームアプリ」のMVPを作成する。

このアプリは、館山・沖ノ島で21人程度の大人数旅行を行う際に、チーム対抗のサバイバルゲーム、物資争奪、ミッション、沖コイン、オークション、チーム再編成を管理するためのWebアプリである。

App Storeに出すネイティブアプリではなく、URLを共有し、スマホブラウザで開き、必要に応じてホーム画面に追加して使えるPWAとして実装する。

## 2. 最重要コンセプト

このゲームは、単なるチームポイント制ではなく、「個人が沖コインを持ち、イベントごとにチームが再編成される」ゲームである。

そのため、沖コインは原則としてチームではなく個人に紐づける。

チームの強さは、その時点で所属しているメンバーの沖コイン合計によって計算する。

## 3. 想定ユーザー

* 幹事
* 参加者
* 参加人数：約21人
* チーム数：3〜4チーム程度
* 1チーム：5〜7人程度

## 4. 技術スタック

* Next.js
* TypeScript
* Tailwind CSS
* PWA対応
* localStorageによる状態保存
* 後からSupabaseに移行しやすい設計

## 5. 実装方針

最初は外部DBなしで完成させる。

状態管理はlocalStorageでよい。

ただし、後からSupabaseへ移行しやすいように、型定義、データ操作関数、UIコンポーネントを分離する。

データアクセスは直接localStorageを各画面で触るのではなく、repository層またはstorage utilityを経由する。

# AGENTS.md

## プロジェクト概要

このリポジトリは、館山・沖ノ島で21人程度の大人数旅行を行う際に使用する、スマホ向けPWA「沖ノ島サバイバル」の開発プロジェクトです。

App Storeに出すネイティブアプリではなく、URL共有で使えるNext.js製PWAとして実装します。

参加者はスマホブラウザでURLを開き、必要に応じてホーム画面に追加してアプリのように利用します。

## 必ず参照するドキュメント

実装前に以下のドキュメントを必ず読んでください。

- docs/REQUIREMENTS.md
- docs/GAME_DESIGN.md
- docs/DESIGN_GUIDE.md
- docs/PWA_AND_NOTIFICATION.md
- docs/DEVELOPMENT_PLAN.md

## 技術スタック

- Next.js App Router
- TypeScript
- Tailwind CSS
- localStorage
- PWA
- 将来的にSupabaseへ移行しやすい設計

## 最重要設計思想

このアプリは単なるチームポイント管理アプリではありません。

「個人が沖コインを持ち、イベントごとにチームが再編成される」ゲームです。

そのため、沖コインは原則としてチームではなく個人に紐づけます。

チームの合計沖コインは、所属メンバーの沖コイン合計から計算してください。

## 実装方針

- まず動くMVPを優先する
- UIよりもゲームロジックの正しさを優先する
- 型定義は lib/types.ts にまとめる
- 初期データは lib/seed.ts にまとめる
- localStorage操作は lib/storage.ts に集約する
- ゲームロジックは lib/gameLogic.ts などに分離する
- 画面コンポーネントとロジックを混ぜすぎない
- 後からSupabaseに移行しやすい構造にする

## MVPで必須の機能

- メンバー管理
- 個人ごとの沖コイン管理
- チーム数指定
- ランダムチーム編成
- チーム再編成
- チーム合計沖コイン表示
- ミッション一覧
- ミッション発動
- ミッション達成承認
- コイン付与
- アイテム付与

- 管理者画面

- アプリ内通知
- PWA対応

## 後回しでよい機能

- Supabase完全移行
- 本格Web Push通知
- 同盟申請
- 交渉機能
- チャット
- 複雑な妨害カード
- GPS常時監視
- AI写真判定

## デザイン方針

デザインの参考イメージとして、サマーウォーズに登場するOZのような、白基調でカラフルな仮想空間・オンライン広場・未来的なゲームロビーの雰囲気を参考にしてください。

ただし、OZのロゴ、キャラクター、固有UI、画面構成、具体的なビジュアル表現を直接コピーしないでください。

あくまで以下の抽象的な方向性として取り入れてください。

- 白を基調にした明るい仮想空間
- カラフルなチームカラー
- 浮遊感のあるカードUI
- オンラインゲームのロビー感
- 未来的だが親しみやすい雰囲気
- 大人数が集まるデジタル広場
- アバターやチーム表示が映えるUI
- 現実の沖ノ島イベントを、仮想世界上のゲームとして進行しているような世界観

## ゲームバランス方針

- コインは個人管理
- チーム報酬はメンバーへ均等配分
- チーム支払いはMVPでは人数割り
- 妨害カードは3種類程度に抑える
- 妨害効果は弱めにする
- 強すぎる没収や奪取は禁止
- 管理者が手動でコイン調整できるようにする
- 重要操作はイベントログに残す

## 1. ゲームコンセプト

このゲームは、館山・沖ノ島旅行を「現実世界で行うサバイバルゲーム」として扱うためのものです。

重要なのは、チームは固定ではなく、イベントごとに再編成される可能性があることです。

そのため、沖コインはチームではなく個人が持ちます。

# PWA_AND_NOTIFICATION.md

## 1. PWA方針

このアプリはネイティブアプリではなくPWAとして作成します。

参加者はURLをSafariまたはChromeで開き、必要に応じてホーム画面に追加して使います。

## 2. PWA要件

- manifest.jsonを用意
- name：沖ノ島サバイバル
- short_name：沖サバ
- display：standalone
- theme_colorを設定
- background_colorを設定
- icon-192.pngを用意
- icon-512.pngを用意
- service workerを用意
- ホーム画面追加できる構成にする

## 3. iPhoneでの使い方

iPhoneでは以下の手順で使います。

1. SafariでURLを開く
2. 共有ボタンを押す
3. 「ホーム画面に追加」を押す
4. ホーム画面に追加されたアイコンから起動する

## 4. 通知方針
